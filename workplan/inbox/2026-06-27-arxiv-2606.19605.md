The npm run prompt-lab -- autotune workflow currently relies on human-directed iteration over prompts/tutor-ego*.md . FAPO's step-level attribution + autonomous loop is a drop-in pattern for this: treat each turn of the tutor pipeline (ego generate → superego critique → ego revise) as a named stage, attribute low tutor_first_turn_score returns to the correct stage, then mutate only that stage's prompt. Especially valuable for cells 93–100 (superego variant ablations) where the failing stage is already known but the correct prompt edit is not.
