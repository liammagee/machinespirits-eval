## The important distinction: no signal vs no valid testbed

The Codex result has two possible readings.

The conservative reading is:

> There is no evidence that the ownership/didactic layer improves anything over hidden+proofDebt, so stop.

The more precise reading is:

> The current artifact pool is not a clean enough testbed for ownership gains. Proof-safe pairs lack ownership movement, and ownership-moving pairs are confounded by release/proof mismatch.

The second reading is the one I would use. When ŌĆ£higher-ownershipŌĆØ examples are disqualified because proof/release changed, that is not a near-win. It is exactly the confound the new evaluator was supposed to catch. It means ownership gains are currently entangled with proof-control changes.

## What is missing now

The missing artifact is a **calibrated ownership benchmark**, not another runtime layer.

It should have three kinds of cases.

First, **positive controls**, where S1 should obviously score higher if the evaluator works. For example, S0 has the learner echo `p_point`; S1 asks for a teach-back and the learner gives an own-words account. Same release schedule, same proof status, same target object.

Second, **negative controls**, where prose is nicer but ownership is unchanged. This prevents the scorer from rewarding warmth, verbosity, or dramatic texture as if it were learning.

Third, **disqualification controls**, where ownership appears higher only because proof-control changed. Codex already found these in the wild; now make them explicit and ensure the scorer rejects them every time.

Until those controls pass, the current ŌĆ£no qualifying success caseŌĆØ should be read as a valid negative on the artifact pool, not yet as a global negative on ownership adaptation.
