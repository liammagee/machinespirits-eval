STAGE: [The learner is looking at a tiny code snippet with a function definition and one function call. The parameter name appears in the definition; the argument value appears in the call. The immediate task is only to label those two roles.]

STAGE: [A clear sleeve holds one printed line, `function greet(name)`, above a second line, `greet("Maya")`, with two blank adhesive labels waiting at the margin.]

LEARNER: [slides the worksheet so both code lines are visible]

"I marked `name` as the placeholder on the definition line, but `"Maya"` on the call line keeps pulling me back. Is `"Maya"` only the argument, not the parameter?"

TUTOR: [draws a short divider between the definition line and the call line, then taps each side with the pencil]

"Definition side: function name, parameter `name`. Call side: function name, argument `"Maya"`. The pull is real because they sit in the same slot, but the label is by side: mark `name` as parameter and `"Maya"` as argument, then read those two labels back once before checkoff."

LEARNER: [keeps the pencil on the divider]

"The divider gives the rule: definition side, `name` gets parameter; call side, `"Maya"` gets argument. The same slot still pulls my eye, so position alone is the trap. For checkoff, `name` is the placeholder parameter, and `"Maya"` is the value supplied later."

TUTOR: [adds two small checkoff boxes under the lines: parameter / argument]

"Function name repeats; parameter and argument do not. Same slot is the trap; line role decides the label. Use that rule now: definition line first, call line second. Put one check beside each correct label, then slide the divider away for checkoff."

STAGE: [The sleeve is slid down so only `function greet(name)` is visible; the `greet("Maya")` line remains covered but its blank label stays on the desk.]
[The latest object-check does not settle cleanly; the marked answer and the remaining task now sit under visible pressure.]

LEARNER: [checks the parameter box beside `name` and leaves the argument box blank]

"`name` is the definition-side parameter; that mark is clean. The covered call line keeps the second box untested, because the matching slot is hidden. Slide the sleeve to `"Maya"` and the argument mark can be checked there."

TUTOR: [slides the sleeve only to expose `"Maya"` and sets a narrow card below it: `names slot` / `fills slot`]

"The checkbox route settled `name`; it does not settle the same-slot pull. Use the criterion now: parameter names the slot on the definition line; argument fills the slot on the call line. Mark `"Maya"` under `fills slot`, then read the release line: `name` names the slot; `"Maya"` fills it."

LEARNER: [marks `"Maya"` under `fills slot` and leaves the divider in place]

"`"Maya"` fills the slot; `name` names it on the definition line. The same position still pulls my eye, but this exposed token passes the `fills slot` test, so the argument check can stand."
