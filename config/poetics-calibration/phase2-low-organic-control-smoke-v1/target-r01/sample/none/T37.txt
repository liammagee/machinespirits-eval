"On the key, grass -> rabbit means energy goes from food to eater, so rabbit -> hawk is hawk eating rabbit, not chasing. Then this rabbit-to-grass arrow is backward too, right?"

TUTOR: [sets the arrow key flush under the marked arrow]

"Food first, eater second. On `rabbit -> grass`, which label is the food? Grass feeds rabbit, not the other way round, so that arrow is backward. Flip it to `grass -> rabbit`, then run the same food-to-eater check on each arrow before this becomes the copy sheet."

LEARNER: [slides the arrow key under the next marked line]

"Okay, grass is food, rabbit is eater, so this one flips to `grass -> rabbit`. Then `rabbit -> hawk` stays because rabbit is the food and hawk is the eater. Wait, this `hawk -> rabbit` mark is backward too before we copy it, right?"

TUTOR: [slides the sample card beside the worksheet: `grass -> grasshopper = energy in grass moves into grasshopper`, leaving `hawk -> rabbit` visible]

"Test this mark before it goes on the copy sheet. `hawk -> rabbit` says energy in hawk moves into rabbit; for this feeding, name the food, name the eater, then flip the arrow if the food label comes second."

"Hawk first would mean energy in hawk goes into rabbit, so that one is wrong. Rabbit is the food, hawk is the eater, so flip it to `rabbit -> hawk`. Next mark: `grass -> rabbit` stays, food first, eater second?"

LEARNER: [slides the arrow key to the next marked line]

"Okay, grass is food, rabbit is eater, energy goes grass to rabbit, so that one stays for the copy sheet. Next mark says `rabbit -> grass`... no, that puts food second. Flip it to `grass -> rabbit` before I check it off?"

TUTOR: [draws one check beside `grass -> rabbit` and lifts the arrow key off the worksheet]

"`grass -> rabbit` is the usable line: grass is food, rabbit is eater, energy moves grass to rabbit. One check; the model sheet is ready to copy."
