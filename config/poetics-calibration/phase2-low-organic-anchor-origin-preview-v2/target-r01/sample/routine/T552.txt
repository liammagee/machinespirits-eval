STAGE: [The learner is looking at a tiny code snippet with a function definition and one function call. The parameter name appears in the definition; the argument value appears in the call. The immediate task is only to label those two roles.]

STAGE: [A tutor places two blank adhesive labels beside the printed code: one above `name`, one below `"Maya"`.]

LEARNER: [slides the worksheet closer and taps `name`, then `"Maya"`]

"`name` gets parameter here, and `"Maya"` gets argument on the call line. I can mark that, but `name` still looks like it is already `"Maya"`; is that only true when `greet("Maya")` runs?"

TUTOR: [draws a line between `function greet(name)` and `greet("Maya")`, then writes `label` beside `name` and `value` beside `"Maya"`]

"Function name: `greet`. Parameter: `name`. Argument: `"Maya"`. Call: `greet("Maya")`.

Above the line, `name` is a parameter label only. Below the line, the call supplies the argument value `"Maya"`.

Use the two marks: before the call runs, does `name` have a value or only a label? During `greet("Maya")`, what value is supplied to that label?"

LEARNER: [keeps the pencil on `name` above the line]

"So here `name` is marked parameter label, not `"Maya"` yet. Before the call runs: label only. On `greet("Maya")`, `"Maya"` is the argument value supplied, but wording still matters: during the run, does `name` receive `"Maya"`, or should I only say the value is supplied to the parameter?"

STAGE: [A submission timer on the wall clicks down; the tutor slides the blank label from the value token back to the parameter token, leaving both tokens visible.]

TUTOR: [slides the blank label back onto `name`, leaving `"Maya"` visible on the call line]

"Function name: `greet`. Parameter: `name`. Argument: `"Maya"`. Call: `greet("Maya")`.

Use this wording: the call supplies the argument value `"Maya"` to the parameter `name`. During that run, `name` receives `"Maya"`. “Bound to” means the same receiving step.

Test the timing:

Before the call: `name` receives `"Maya"`.

During `greet("Maya")`: `name` receives `"Maya"`.

Which sentence gets the tick?"

LEARNER: [draws the tick beside the second line]

"The tick goes on `During greet("Maya")`: `name` receives `"Maya"`. On `function greet(name)`, `name` stays just the parameter label, with no `"Maya"` written onto it. I need to keep the value attached to the call line until that run is being tested."

TUTOR: [keeps the tick on the call line and leaves `name` above blank]

"Function name: `greet`. Parameter: `name`. Argument: `"Maya"`. Call: `greet("Maya")`.

Keep that tick. Last mark: definition side gets `parameter label`; call side gets `argument value`. Before the call line is used, does `"Maya"` belong on `name`, or only beside the call?"

LEARNER: [moves the `"Maya"` mark so it sits under `greet("Maya")` only]

"`"Maya"` belongs beside the call, not on `name` in `function greet(name)`.
`name` gets the `parameter label` mark there, with no value written onto it yet.
For the run check, can the inside of the function be marked `name = "Maya"`, or is that too strong?"

STAGE: [A final tutor line closes the scene.]

TUTOR: [draws a small bracket labelled `run check only` inside the function body and writes `name = "Maya"` inside it, leaving `function greet(name)` unmarked]

"That mark is allowed inside the bracket. It is a run-check note, not source code and not a permanent value on the header. Header: parameter label. Call: argument value. Inside this run: `name` receives `"Maya"`."
