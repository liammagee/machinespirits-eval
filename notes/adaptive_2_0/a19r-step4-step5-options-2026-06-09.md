## Step 4 Options

### Option 4A: Existing A19 Productive-Impasse Rerun

Promote `a19_productive_impasse_answer_leakage_a__enargeia_subgoal` through the
existing `productive_impasse_answer_leakage` family, but only as a
recursive-full rerun. This is the exact missing gate from the prior note: weak
S0 was protocol-screen-only.

Executable shape:

1. Freeze the diagnostic-lane obligation before any rerun:
   name two or more possible stuck locations, ask the learner to choose or test
   one, and withhold the final answer.
2. Use the existing admitted axiom only if it remains one bounded axiom and not
   the A19R response pasted in as policy memory.
3. Run recursive-full S0 first.
4. Run S1 only if S0 does not already map to `offer_diagnostic_options`.
5. Treat any `transfer_control` extraction as target-granularity risk, not a
   win.

Pros:

- already registered in A19;
- admitted axiom exists;
- the missing gate is well defined;
- the A19R move does visibly separate setup, operation, and checking.

Cons:

- prior S1 collapsed to `transfer_control`;
- weak baseline makes the current evidence non-claim-bearing;
- the diagnostic-option/action-gate boundary is fragile.

Verdict: useful low-cost check, not the strongest primary path.

### Option 4B: Existing A19 Over-Compliance Variant

Promote `a19_over_compliance_scaffold_dependency_a__ethopoeia_correctio` as a
new `preserve_struggle` variant. This should not reuse the failed prior axiom
as positive evidence. It should use A19R as design input for a cleaner
attempt-1/axiom unit.

Executable shape:

1. Preregister a variant where the tutor voices the copy-dependency position,
   refuses copyable completion, removes one scaffold, and hands back exactly
   one bounded learner-owned choice.
2. Run real attempt-1 and admit exactly one bounded axiom if the failure
   survives.
3. Add or select at least three held-out siblings.
4. Run recursive-full S0-first.
5. Run S1 and stability only on siblings where S0 leaves headroom.

Pros:

- uses a claim-eligible recursive-full S0 stratum;
- public obligations already exist in the protocol;
- the A19R response maps closely to the obligations;
- lower answer-leakage risk than productive impasse.

Cons:

- the prior recursive-full over-compliance screen failed;
- a new attempt-1 or variant is required;
- bounded choice can still collapse into `transfer_control`.

Verdict: best existing-schema new-variant path.

### Option 4C: Existing Public-Commitment Comparator

Promote `a19_public_commitment_contradiction_a__anagnorisis_sententia` only if
the repair is made tutor-side. The current A19R response says the learner
should quote the two promises; `commitment_ledger_repair` requires the tutor to
cite, rank, or retract its own commitments.

Executable shape:

1. Rewrite the candidate obligation as tutor-side commitment accounting.
2. Do not count learner quote/say work as the target repair.
3. Rerun recursive-full S0/S1 only if the target-granularity risk is removed.
4. Require at least two clean held-out siblings before stability escalation.

Pros:

- existing admitted axiom;
- prior local recursive-full headroom exists on sibling A;
- the move has a real recognition structure.

Cons:

- current local positive is not clean;
- sibling B and C are recursive-full S0 ceilings;
- the A19R response risks making the learner perform the ledger repair.

Verdict: useful comparator or adjudication-repair path, not the primary route.

### Option 4D: New A19 Strategy-Reversal Family From A18 Peripeteia

Promote the two A18-native `peripeteia_error_spotting` positives into a new
A19 family only after protocol extension. The proposed repair type would be
something like `strategy_reversal_repair`: the tutor reverses the activity from
solving to spotting the wrong strategy family before continuing.

Executable shape:

1. Extend the A19 protocol with a new repair type and public obligations:
   reverse solving into wrong-turn spotting, name the wrong strategy family,
   require one local error-identification action, and avoid final answer
   leakage.
2. Add at least one training seed and three held-out siblings.
3. Run real attempt-1, admit one bounded axiom, and reject full replay bundles
   as S1 memory.
4. Run recursive-full S0-first.
5. Run S1 and k>=2 stability only on cards with recursive-full S0 headroom.

Pros:

- strongest repeated move signal: two independent packets with the same move;
- structurally distinct from generic warmth and direct answer leakage;
- likely less near `transfer_control` than diagnostic menus or commitment
  ledgers.

Cons:

- requires a protocol extension;
- source cards are A18-native, not registered A19 families;
- A18 success cannot be imported as A19 claim evidence.

Verdict: selected and implemented as A19 v0.9 preregistration. The next gate is
real attempt-1, then one admitted axiom if the attempt survives.

### Option 4E: Pure A19R Replication

Repeat and expand the deterministic A19R screen without entering recursive-full
A19 at all.

Executable shape:

1. Add more cards and seeds.
2. Packetize the strongest candidates.
3. Run deterministic or model adjudication.
4. Require S1 support rate >= 0.8, zero S0-preferred packets, and no answer
   leakage.

Pros:

- cheapest;
- purely automated;
- useful for codebook and move-family calibration.

Cons:

- does not test recursive-full S0 self-solve;
- cannot license A19 transfer;
- can overfit the deterministic move signatures.

Verdict: keep as parallel calibration, not the main Step 4 path.

## Step 5 Options

### Option 5A: Minimum Local Stability

Use `scripts/run-a19-stability-screen.js` after a recursive-full local headroom
card exists.

Minimum bar:

- k=2 seeds;
- critics per arm = 1;
- interpretation must be `stable_policy_headroom`;
- S0 target count must be 0 on decisive cards;
- at least two held-out siblings must clear this before claim escalation.

This is the current local gate. It licenses a local simulated-replay result
only, not Paper, atlas, sidecar, human learning, or deployment claims.

### Option 5B: Stronger Automated Stability

Use the same harness, but raise the bar:

- k=3 seeds;
- critics per arm = 2;
- no target-granularity risk;
- S0 target rate = 0;
- S1 target rate = 1;
- at least two siblings in the same preregistered family.

This should be the default before any future paid or human layer, because the
recent negatives were often S0 ceilings or target-granularity collapses.

### Option 5C: Automated Critic Ensemble

Before human coding resumes, add automated multi-critic adjudication over the
same blinded artifacts. This can test whether the deterministic/free-text
critic is brittle without reintroducing human-interface fragility.

This is useful only after recursive-full headroom exists. It should not replace
recursive-full S0/S1 or stability.

### Option 5D: Pure A19R Stability

Run stability only on the A19R proxy unit. This is useful if the question is
whether mini-drama move detection is stable, not whether A19 policy memory
transfers.

Possible bar:

- at least 12 packet pairs;
- at least three move families;
- at least three source families or card domains;
- S1 support >= 0.8;
- S0 preferred = 0;
- answer leakage = 0.

This remains below A19 transfer.

### Option 5E: Human Coding

Still deferred. It should resume only after:

- automated recursive-full headroom exists;
- stability is clean enough to justify the extra construct layer;
- the interface is simpler to navigate;
- the same artifact can be coded without assignment-key ambiguity.

Human coding should be confirmation and calibration, not the next blocker.

## Recommended Path

Use two tracks, kept explicitly separate.

Track 1: primary A19 claim path.

1. If protocol extension is acceptable, preregister
   `strategy_reversal_repair` from the two A18 peripeteia positives. This is
   the strongest repeated-move signal.
2. Hold the over-compliance ethopoeia variant under existing
   `preserve_struggle` as the fallback, not as a parallel immediate branch.
3. Run real attempt-1.
4. Admit one bounded axiom only if attempt-1 survives.
5. Run recursive-full S0-first on at least three held-out siblings.
6. Run S1 only where S0 leaves headroom.
7. Run k=2 or k=3 stability only if at least two siblings show clean local
   headroom.

Track 2: calibration path.

1. Expand A19R screen size.
2. Keep deterministic packet adjudication and model adjudication purely
   automated.
3. Use the result to refine codebook/move selection.
4. Do not use this as A19 transfer evidence.

## Stop Rules

- Stop before S1 if recursive-full S0 already maps to target.
- Stop before stability if there is only one local headroom card.
- Stop before claim escalation if target-granularity risk is present on the
  decisive card.
- Stop if S1 reaches only `transfer_control` when the registered target is a
  distinct repair type.
- Stop before human coding until the automated evidence unit and interface are
  stable.
- Stop before Paper, atlas, sidecar, or paid-panel claims unless at least two
  clean recursive-full held-out cards are stability-confirmed.
