P2 · Rubric dimensionality audit: run calibrate-rubric.js and cross-walk against KMP-Bench / 2606.16206

Papers: 2606.08625 (rubric survey — dimensionality collapse warning), 2606.16206 (teach-or-solve gap, MathTutorBench, r = 0.42), 2603.02775 (KMP-Bench: 6 pedagogical dimensions)
Why now: 2606.08625 maps the failure modes of multi-dimension rubrics — dimensionality collapse (nominally-many dimensions that converge in practice to one) is their primary warning. The project now carries three independent rubric families (v2.2 tutor 8-dim, charisma v1.0 8-dim, poetics v1.0 6-dim). 2606.16206 independently confirms that the teach-vs-solve split matters (the central design choice in v2.2) and benchmarks it against MathTutorBench; their solve/teach correlation of r = 0.42 is the external number to anchor the §5.4 rubric-consistency discussion. KMP-Bench (2603.02775) offers 6 named pedagogical dimensions (Challenge, Explanation, Modelling, Practice, Questioning, Feedback) that can be used as an external validity lens on v2.2's 8 dimensions.
Do: Run node scripts/calibrate-rubric.js on existing scored runs to extract inter-dimension Pearson correlations for the v2.2 8-dimension tutor rubric and the 6-dimension poetics rubric. Flag any dimension-pair with |r| > 0.85 as a collapse candidate. Then manually map v2.2's 8 dimensions against KMP-Bench's 6 and note which v2.2 dimensions have no KMP-Bench analogue (those are the project-specific additions, needing their own internal-validity defence). Write findings into docs/research/paper-full-2.0.md §5.4 (rubric consistency) with the collapse-candidate table and the KMP crosswalk. If any dimension consistently correlates > 0.85 with another, flag it as a candidate for merging in a hypothetical v2.3 (a human decision, not to implement now).
Evaluate: Existing data — run calibrate-rubric.js (pure computation, no API calls) on the existing data/evaluations.db. Metric: inter-dimension Pearson matrix; collapse threshold |r| > 0.85. Target: confirm that content_accuracy does not collapse into the scaffolding dimensions (as 2606.16206 predicts it should not if the split is real). existing data — no paid eval

P3 · Pilot reasoning-synthesis aggregation for the poetics critic panel (κ ≈ 0.04 fix candidate)

Papers: 2606.11609 (reasoning-level synthesis for multi-agent stance detection), 2606.15783 (SemEval Task 4: multi-view narrative similarity)
Why now: The poetics arc phase-2 finding — weighted κ ≈ 0.04 vs ≥ 0.60 target — was treated as a failure of the instrument (the rubric classifies dramatic form, not mind-reading). But 2606.11609 shows that label-level aggregation of rationale-producing critics is systematically worse than synthesising their reasoning traces through a coordinator step. Our critics already output free-text reasoning (scores_with_reasoning); we may be aggregating at the label level (poetics:audit computes κ on scores) when we should synthesise the rationales first. 2606.15783's multi-view encoder (separate heads for theme, plot, outcome) maps almost exactly onto the three-act dramatic structure and could provide a more principled per-dimension comparison framework for the inter-critic audit.
Do: Pull three existing critic outputs for a sample of ≥ 10 transcripts from exports/phase2-classic-drama-*. For each transcript, extract the reasoning prose from scores_with_reasoning alongside the numeric scores. Run a coordinator prompt that reads all three critics' reasoning traces and synthesises a consensus label (separate from each critic's individual label). Compute κ under both aggregation strategies (label-average vs reasoning-synthesis) and compare. Log results in a new note notes/poetics/2026-06-22-critic-aggregation-pilot.md. If κ improves materially (> 0.20), draft the mechanism change for poetics:audit as a separate PR proposal (human to authorize the code change).
Evaluate: Existing exports/phase2-classic-drama-* data — no new generation needed if scores_with_reasoning was captured. The coordinator synthesis step requires one API call per transcript-batch (small, ~10 transcripts × 1 call). Metric: weighted κ under reasoning-synthesis vs label-average; secondary metric: variance in synthetic consensus score. ~10 API calls (coordinator synthesis); otherwise existing data
