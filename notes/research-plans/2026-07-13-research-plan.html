The plan

P1 · Audit the judge pipeline for contagion with "Contagion Networks: Evaluator Bias Propagation in Multi-Agent LLM Systems" (2606.20493)

Why: formalizes how one agent's biased judgment contaminates downstream agents that consume it as context, and proposes independent sampling and adversarial-critic insertion as mitigations. This is a direct challenge to an assumption the project's own inter-rater methodology depends on: analyze-judge-reliability.js and the paired-rejudge protocol (CLAUDE.md → "Inter-Rater Reliability") treat judges as independent. It is also a structural challenge to the Ego→Superego→Ego loop itself — a biased superego critique can compound across revision rounds exactly the way the paper's "contagion" propagates across an agent chain, which is what §5.4's systematic-drift checks are trying to catch from the output side rather than the mechanism side.
Do: two read-only checks, no new cell. (1) Code audit: trace the judge invocation path in services/evaluationRunner.js / scripts/eval-cli.js evaluate and the rejudge path to confirm judge prompts never include another judge's prior verdict or a superego's private critique as context — i.e. rule out a literal contagion channel, not just a statistical one. (2) Re-analyze existing paired-rejudge data with analyze-judge-reliability.js for a contagion signature — specifically, do disagreements cluster by which cell's superego framing produced the transcript (22-33 divergent-superego family) rather than being uniform across cells — the pattern the paper's network model predicts if superego bias is leaking into judged output rather than judges disagreeing independently.
Evaluate: existing rejudged pairs already in the DB (judge_model-partitioned rows for any run with a rejudge pass) — extend analyze-judge-reliability.js's correlation output with a per-cell-family breakdown. No new generation or judging. existing data

P2 · Cross-walk the two-pass/coupling superego cells against MACR explicit conflict resolution (2606.20245)

Why: MACR rejects the binary "trust the model or trust the context" framing and instead surfaces ego/context conflicts explicitly, adjudicating them through a distinct third step rather than folding resolution into a single revision pass. Cells 98 (cell_98_base_dialectical_suspicious_unified_two_pass) and 99 (cell_99_base_dialectical_coupling_unified_superego) — the two-pass and coupling variants of the dialectical-suspicious superego ablation family (93-100) — currently resolve the ego/superego conflict within a single revision, which is exactly the pattern MACR argues under-adjudicates.
Do: no new cell yet. Re-analyze existing tutor_deliberation_* logs for cells 98 and 99 against the 93-100 family baseline, checking whether incorporation_rate / dimension_convergence already show a measurable "under-adjudication" signature (e.g. superego points raised but neither accepted nor explicitly rejected in the ego's revision) — the read-only precursor that would justify a MACR-style explicit-adjudication cell (a cell_126+ candidate, out of scope for this review) rather than speculatively proposing one.
Evaluate: existing dialogue logs for cells 93-100, no new generation — a new analysis script over tutor_deliberation_*, incorporation_rate, dimension_convergence. existing data

P3 · Check whether recognition prompts state Hegelian dynamics as structure or prose, against Equation-to-Behavior Prompting (2606.17657)

Why: the Princeton persuasion-simulation paper finds that encoding a theoretical specification as explicit equations/structural rules in the prompt — not prose description — makes larger models reproduce the target behavior more faithfully across a broader range. This is a direct methodological twin to how recognition-mode prompts encode Hegelian mutual-recognition theory today.
Do: read-only prompt audit first, not an autotune run: diff the recognition-mode ego/superego prompts (the base cells 5-8 and messages-mode cells 84-90) against the placebo/base prompts using npm run prompt-lab -- diff, and characterize how much of the recognition content is stated as prose narrative ("the tutor should recognize the learner as an autonomous subject...") versus explicit structural rule/condition. If it's predominantly prose, that's the concrete gap 2606.17657 predicts matters — a candidate input to a future prompt-lab -- fork + autotune experiment on cells 5-8, which would be a paid run and is out of scope here.
Evaluate: prompt-lab -- diff / direct reading of prompts/tutor-ego-recognition*.md against prompts/tutor-ego-placebo.md — no model calls. existing data
