Prompt Engineering

MAS-PromptBench: When Does Prompt Optimization Improve Multi-Agent LLM Systems? UNBLOCK

Juyang Bai, Laixi Shi (Johns Hopkins University) · arXiv 2606.23664 · June 23, 2026 · https://arxiv.org/abs/2606.23664

A benchmark spanning diverse tasks, workflow topologies, communication protocols, team sizes, and optimizers (including MAS-GEPA and MAS-MIPRO), built specifically to test when prompt optimization helps vs. hurts multi-agent LLM systems. Gains run as high as 24 points but losses run as low as -16 points on the same tasks; optimization works best when agent-level behaviors are explicit/verifiable and communication protocols have shared structure, and gets harder as team size (coordination overhead) grows.

Project relevance: Directly bears on npm run prompt-lab -- autotune and any future automated tuning of prompts/tutor-ego*.md / tutor-superego*.md: the bilateral ego-superego architecture is exactly a 2-agent MAS with an explicit communication protocol (critique → revision), which is the regime MAS-PromptBench finds optimization works best in. The negative-gain finding is a caution for cells 93-100 (superego variant ablations) — before autotuning, check whether the superego's critique surface is "explicit and verifiable" enough for optimization to help rather than regress cell scores.

Agent Architecture & Ontology

From Tokens to States: LLMs as a Special Case of World Models and the Continuous Path Beyond WATCH

Paul Dubois · arXiv 2606.28127 · June 26, 2026 (opinion/position paper, draft) · https://arxiv.org/abs/2606.28127

Argues against LeCun's binary framing of "token-predicting LLMs" vs. "world-model architectures": LLMs are recast as a degenerate special case of world models, where the state space is the set of all token sequences and the only action is appending a token — making world models a strict generalisation, not a replacement. Proposes a continuous spectrum from next-token prediction to JEPA-style latent prediction, with multi-token prediction, future-summary prediction, and next-latent prediction as intermediate points already populated by current research.

Project relevance: A useful conceptual frame for the id-director architecture (cells 101-109) and the adaptive runner's externalised learner state (services/adaptiveTutor/stateSchema.js): both already treat "state" as a first-class object distinct from raw token history, which is precisely the generalisation this paper argues LLM-native architectures are converging toward. Also a candidate citation for framing why runner: adaptive cells (110-125) outperform pure dialogue-engine baselines — they move state tracking out of the token stream and into an explicit schema, which is the paper's core distinction.

Dramaturgy / Narrative

CASPER in the Machine: Insights into Character Variety in LLM-Generated Stories UNBLOCK

Anneliese Brei, Abhisheik Sharma, Nicholas Sanaie, Lu Wang, Snigdha Chaturvedi · arXiv 2606.22454 · June 2026, accepted ACL 2026 (exact day not independently confirmable — direct arXiv fetch is blocked by this environment's egress policy; arXiv-ID sequencing places it between the June 16 and June 23 papers already dated in this folder, i.e. inside the window) · https://arxiv.org/abs/2606.22454

Builds a controlled corpus of 4,400 LLM-generated stories (7 models, 4 model families), each paired with a human-written story sharing the same writing prompt, and introduces CASPER, an 8-category-pair scheme for classifying character portrayal. Key findings: LLM-generated stories more often give characters completed storylines and explicit growth arcs than human-written ones ("play it safe" with tidier arcs); character diversity does not scale with model size (Phi family most diverse, Llama least, among the 4 families tested); and genre systematically shapes character type (domestic stories skew toward realistic theme-bearing characters, romance toward straightforward ones).

Project relevance: Directly actionable for scripts/drama-generator.js and the poetics rubric. The "tidier arcs / completed storylines" finding is a plausible mechanism behind the phase-2 transfer failure (weighted κ ≈ 0.04) noted in DRAMATIC-RECOGNITION-PLAN.md: if generated tutoring dramas systematically over-resolve character arcs relative to human-authored drama, critics trained on real dramatic form would diverge from critics scoring generated transcripts for exactly the reason CASPER documents. The CASPER category-pairs (stylized/natural, coherent, complex, dynamic, closed, etc.) are also a ready-made secondary coding scheme to cross-tabulate against config/evaluation-rubric-poetics.yaml's peripeteia/anagnorisis dimensions.
