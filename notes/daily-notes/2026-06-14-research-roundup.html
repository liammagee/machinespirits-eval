Pedagogy & Tutoring

SocialCoach: Personalized Social Skill Learning with RL-based Agentic Tutoring and Practice UNBLOCK

[Authors per arxiv page] · arXiv 2606.04155 · June 4, 2026 (first coverage in this note) · https://arxiv.org/abs/2606.04155

SocialCoach is a full-stack LLM-powered agentic tutoring system for social skill development that addresses the knowing-doing gap — when a learner understands a concept but cannot act on it. The system integrates: (1) immersive, goal-driven practice scenarios; (2) causality-driven proficiency assessment that goes beyond surface correctness; and (3) knowledge-grounded reflective tutoring with an adaptive scheduling module (prescription → retrieval → adaptation) whose policy is optimised in a learner simulation environment via RL. Deployed in the EQoach product; judge-rated tutoring quality exceeds baselines.

Project relevance: The "knowing-doing gap" is the same phenomenon tracked by analyze-insight-action-gap.js (Finding 11 / D3): tutor rhetoric that recognises the learner's position without changing the learner's behaviour. SocialCoach's prescription-retrieval-adaptation loop is structurally identical to the adaptive runner's trap-trigger → policy-action → counterfactual-replay cycle. The causality-driven proficiency assessment is a methodological precedent for upgrading the learnerGrowthIndex from text-surface proxies to causal attribution.

Evaluation & Rubrics

From Holistic Evaluation to Structured Criteria: Rubrics Across the Evolving LLM Landscape UNBLOCK

Hao Chen et al. · arXiv 2606.08625 · June 7, 2026 · https://arxiv.org/abs/2606.08625 | GitHub: AI9Stars/LLM-Rubrics-Survey

A systematic survey that repositions the rubric as the unifying evaluation primitive across LLM assessment, RLHF, and safety alignment — framing rubrics as a dynamic institutional response to successive paradigm shifts in language model capability. Organises existing rubric designs by construction strategy (manual, LLM-generated, optimised), examines their use in training vs. evaluation contexts, and maps failure modes (dimensionality collapse, judge gaming, cross-task generalization).

Project relevance: The project now maintains three independent rubric families (v2.2 tutor, charisma v1.0, poetics v1.0). This survey is the first comprehensive map of the design space and failure-mode taxonomy for that kind of multi-rubric architecture. The "dimensionality collapse" finding — rubrics that nominally have many dimensions but practically converge to one — is a direct audit target for the 8-dimension v2.2 rubric and the 6-dimension poetics rubric. Also surfaces methodological priors for the calibrate-rubric.js workflow.

Agent Architecture & Social Cognition

GroupToM-Bench: Benchmarking Group Theory of Mind and Nonlinear Social Emergence in MLLMs WATCH

[Authors per arxiv page] · arXiv 2606.04184 · June 4, 2026 (first coverage in this note) · https://arxiv.org/abs/2606.04184

First multimodal benchmark for group-level Theory of Mind, built around a three-tier causal chain: micro-level BDI states (belief, desire, intention) → meso-level group tension and structural constraints → macro-level outcome prediction. A seven-level audit framework assesses from individual attribution (levels 1–3) to nonlinear social emergence that cannot be recovered by summing individual intentions (levels 4–7). Current frontier MLLMs fail comprehensively at levels 4–7.

Project relevance: The bilateral ToM cells (115–123, P2.1) measure exactly this kind of cross-agent mind-reading. GroupToM-Bench provides a vocabulary (BDI × group tension × structural constraint) and a benchmarking methodology that could sharpen the P2.1 state-schema (§6.8.5–.6) beyond the current binary pass/fail on trap-scenario recognition. The "nonlinear emergence" finding — that group-level outcomes are irreducible to summed individual ToM — is also a theoretical grounding for the bilateralTransformationIndex as something genuinely distinct from the sum of adaptationIndex + learnerGrowthIndex.

Less Context, More Accuracy: A Bi-Temporal Memory Engine for LLM Agents WATCH

[Authors per arxiv page] · arXiv 2606.09900 · June 10, 2026 · https://arxiv.org/abs/2606.09900

Proposes a bi-temporal memory layer for LLM agents that tracks (a) when a fact was true and (b) when the agent learned it — the gap between these is the source of distractor accumulation when replaying full history into a prompt. Shows that lean bi-temporal retrieval consistently outperforms full-history replay, with the accuracy gap widening as session length grows.

Project relevance: The ego generates each turn by replaying the entire dialogue context into the prompt. For long multi-turn cells (messages-mode: 80–92, adaptive: 110–113), this replay accumulates "distractor" turns that the ego must filter. Bi-temporal memory is a candidate architecture for the ego's context management and could complement the existing Agent Memory characterization work (2606.06448, covered June 8).

Prompt Engineering

APEX: Automated Prompt Engineering eXpert with Dynamic Data Selection WATCH

Fei Wang, Si Si, Cho-Jui Hsieh, Inderjit S. Dhillon (Google / UCLA) · arXiv 2606.11459 · June 9, 2026 · https://arxiv.org/abs/2606.11459

Addresses the data-efficiency bottleneck in evolutionary automatic prompt optimisation: current methods treat the development dataset as a static benchmark and waste compute on "easy" examples the current prompt already handles correctly. APEX dynamically stratifies examples into Easy / Hard / Mixed tiers based on the optimisation lineage and focuses the evolutionary search on the Mixed tier — cases where the prompt sometimes succeeds, indicating the most leverage.

Project relevance: The prompt-lab autotune workflow optimises tutor/learner prompts against a fixed evaluation set. APEX's Mixed-tier prioritisation directly maps onto the near-miss class of evaluation outputs — scenarios where the ego almost succeeds and where additional prompt signal would be highest-yield. A natural candidate for the next autotune cycle once the poetics arc rubric is stable.
