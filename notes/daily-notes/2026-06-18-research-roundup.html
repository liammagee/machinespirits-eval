Pedagogical AI & Tutoring

Measuring Whether LLM Tutors Teach or Solve: A Diagnostic for Educational Impact UNBLOCK

Junyi Yao, Zihao Zheng, Baichuan Li · arXiv 2606.16206 · June 15, 2026 · https://arxiv.org/abs/2606.16206

The paper isolates a "teach-or-solve gap": stronger task-solving ability in LLMs does not imply stronger learning support. Across eight models on the public MathTutorBench leaderboard, the correlation between solving and pedagogy composites is only r = 0.42, with several models shifting substantially in rank when evaluation switches to the pedagogical dimension. The diagnostic is lightweight: just two benchmark splits plus a gap score.

Project relevance: This is the empirical twin of the project's rubric v2.2 design question. The project already separates content_accuracy from scaffolding dimensions; 2606.16206 provides external validation that this split matters and a ready benchmark (MathTutorBench) for cross-validating cell scores. Could sharpen the §6.3 discussion by quantifying the solve/teach gap across cells — cells with high tutor_first_turn_score but low learner_growth_index are direct instances of the phenomenon.

Prompt Engineering

Environment-Grounded Automated Prompt Optimization for LLM Game Agents UNBLOCK

Rean Clive Fernandes, Lukas Fehring, Theresa Eimer, Marius Lindauer, Matthias Feurer · arXiv 2606.17838 · June 16, 2026 · https://arxiv.org/abs/2606.17838

The paper decomposes an agent's observation-to-action pipeline into named modules (goal descriptor, action selector) and runs an LLM-driven evolutionary loop that (a) attributes episode outcomes to specific module prompts via a behavior analyser, then (b) proposes targeted revisions via a mutator, validated by environment rollouts. The method is fully automatic and achieves meaningful gains without any human re-prompting.

Project relevance: The module decomposition mirrors the ego/superego split almost exactly: the goal descriptor ≈ ego (observation → intention) and the action selector ≈ superego (intention → constrained output). The behavior-analyser + mutator loop is a candidate mechanism for the npm run prompt-lab -- autotune workflow: instead of hand-tuning prompts/tutor-ego*.md, run an offline evolutionary loop guided by tutor_first_turn_score returns. Especially relevant for cells 93–100 (superego variant ablations).

Dramaturgy / Narrative

Fabula: Building a Narrative Storytelling Sidekick with the Writers' Community WATCH

Google DeepMind · arXiv 2606.14411 · June 2026 · https://arxiv.org/abs/2606.14411

Fabula is an interactive narrative-planning sidekick for fiction writers, co-designed with a writers' community and backed by structured LLM outputs (Gemini 3 Flash / Gemini 3.1 Pro). It scaffolds plot coherence through scene-level summaries and character-state tracking, surfacing internal contradictions before they compound.

Project relevance: The drama-generator (scripts/drama-generator.js) and poetics pipeline need exactly this: a coherence scaffold that tracks character state across scenes. Fabula's character-state-tracking approach — maintaining a symbolic envelope of character properties updated scene-by-scene — is a concrete design pattern for extending the drama spec to include per-turn arc annotations, which the evaluation-rubric-poetics.yaml currently leaves implicit.

Ontology / Knowledge Representation

LLMs+Graphs: Toward Graph-Native, Synergistic AI Systems WATCH

arXiv 2606.11560 · June 10, 2026 · https://arxiv.org/abs/2606.11560

The survey maps three emerging LLM–KG synergy patterns: LLMs constructing and curating knowledge graphs, KGs enforcing semantic constraints on LLM output (grounding), and bidirectional co-evolution where each side updates the other. The "graph-native" framing argues that neither side should be a bolt-on — the graph structure should participate in the reasoning loop, not just serve as retrieval.

Project relevance: The poetics ontology (peripeteia / anagnorisis / hamartia / etc.) is currently an informal YAML schema. Promoting it to a proper knowledge graph — and running the poetics critic through a KG-grounded scoring step rather than a free-text rubric — would directly address the κ ≈ 0.04 failure (critics diverging because they lack shared ontological grounding). The bidirectional update pattern also describes how the adaptation-recognition loop should work: each drama run updates the ontology instance, which then constrains the next generation.
