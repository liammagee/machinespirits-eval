Prompt Engineering

SePO: Self-Evolving Prompt Agent for System Prompt Optimization UNBLOCK

(Authors not retrieved — arxiv 403 on fetch) · arXiv 2606.04465 · early June 2026 (gap catch-up) · https://arxiv.org/abs/2606.04465

Existing prompt-agent systems hand-craft and freeze the prompt agent's own system prompt while optimizing task-agent prompts. SePO folds the meta-prompt into the search space: a single agent refines both task-agent prompts and its own system prompt via open-ended evolutionary search over an archive of stepping-stone candidates. Pre-training on a multi-task pool is followed by target-task fine-tuning; gains are shown across math (AIME'25), abstract reasoning (ARC-AGI-1), and code generation (MBPP).

Project relevance: The ego's system prompt in cells 1–8 is currently hand-tuned and frozen at run time. SePO's self-referential design suggests a viable path to jointly optimising the ego prompt and the superego's critique instructions — which is exactly the lever the DRAMATIC-RECOGNITION-PLAN.md §3.2 calls for but leaves unspecified. Worth piloting on a single cell pair (e.g., cells 3/4) before the next large factorial run.

Agent Architecture & Memory

GitOfThoughts: Version-Controlled Reasoning and Agent Memory You Can Replay, Diff, and Merge WATCH

(Authors not retrieved — arxiv 403 on fetch) · arXiv 2606.14470 · ~June 12, 2026 · https://arxiv.org/abs/2606.14470

LLM reasoning chains evaporate with the context window; GitOfThoughts stores an agent's reasoning tree as a git repository — every scored thought is a commit, scores are git notes, outcomes are tags, and retrieval is git log over the agent's own history. Surprisingly, across five memory substrates (none, markdown, vector, graph, git) and two benchmarks, richer memory does not improve accuracy on novel problems, though it greatly aids auditability and cross-agent merge.

Project relevance: The adaptive runner (cells 110–113) already stores a per-turn state trace; the id_construction_trace column holds the id-director arc. GitOfThoughts' null result on accuracy is itself a finding worth noting in §6.8 (adaptive cells don't benefit from richer memory) — but the auditability angle maps onto /ms-replay-one-side's counterfactual replay need: a git-diff of two reasoning branches is a natural representation of the learner-variance vs. structural-cap question.

Adaptive Turn-Taking for Real-time Multi-Party Voice Agents WATCH

(Amazon AGI / IIT Kharagpur) · arXiv 2606.13544 · June 11, 2026 · https://arxiv.org/abs/2606.13544

ModeratorLM conditions a speech-LLM's turn-taking policy on an explicitly assigned moderator role. Combined with chain-of-thought reasoning over conversational context, it improves turn-taking precision >40% and recall >70% versus non-role-conditioned baselines on real meeting data, with a synthetic dataset (RolePlayConv) released for the community.

Project relevance: The multi-turn cells currently implement a strict alternating-turn protocol regardless of content. ModeratorLM's role-conditioned turn policy is a conceptually clean extension for cells where the tutor should hold the floor mid-Socratic-exchange rather than cede to the learner. Low-priority now but directly applicable if the pilot moves to voice/real-time modality (see TODO.md §D).

Dramaturgy / Narrative Ontology

Modeling Narrative Structures via Pseudonymization and Multi-View Sentence Alignment WATCH

Tai Tran Tan, An Dinh Thien (University of Information Technology, Ho Chi Minh City) · arXiv 2606.15783 · June 14, 2026 · https://arxiv.org/abs/2606.15783 | SemEval-2026 Task 4 system paper · GitHub

SemEval-2026 Task 4 benchmarks narrative story similarity: two stories are similar if they share abstract theme, course of action, and outcome, regardless of surface form. This paper's Track B solution learns a multi-view encoder (three separate projection heads for theme, plot, outcome) with self-supervised cross-view alignment, outperforming single-view baselines across all three narrative dimensions.

Project relevance: The poetics pipeline currently scores each transcript on a single holistic rubric (evaluation-rubric-poetics.yaml). The SemEval Task 4 decomposition (theme / course-of-action / outcome) is nearly isomorphic to the three-act dramatic structure the rubric tries to capture. The multi-view alignment approach could provide a more principled way to compare transcripts from different cells on each dramatic dimension independently — a direct route to the inter-cell narrative-distance metric the poetics arc needs but currently lacks. The SemEval dataset itself (released with the task) may also serve as a synthetic training corpus for the critic model.
