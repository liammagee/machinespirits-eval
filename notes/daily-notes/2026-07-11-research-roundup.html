Prompt Engineering

Prompt Coach: An Empirical Evaluation of an Agentic Tutor for Learning Prompt Engineering in Software Development UNBLOCK

Rohit Mehra, Kapil Singi, Vikrant Kaulgud, Vibhu Saujanya Sharma, Swapnajeet Gon Choudhury, Swati Sharma, Adam P. Burden, Majd Sakr · arXiv 2607.06074 · July 2026 · New Ideas and Emerging Results track, ASE '26 · https://arxiv.org/abs/2607.06074

An empirical evaluation of an agentic tutor purpose-built to teach professional software developers prompt engineering itself — the tutor's subject matter is the skill of prompting an LLM, not a downstream domain like fractions or classifier auditing. That reflexive setup (an LLM-tutor teaching people to write prompts for LLMs) is close in spirit to this project's own "recognition" ablations, which manipulate what the tutor's prompt teaches the tutor about itself.

Project relevance: A directly comparable "agentic tutor for a meta-skill" case study, useful as an external benchmark for cells 9-14 (enhanced/hardwired-rules prompt ablations) and cell 21 (dynamic prompt rewriting / Writing Pad): both mechanisms are, in effect, teaching the tutor's own prompt to adapt. Worth reading for its empirical evaluation methodology on whether an agentic tutor measurably improves acquisition of a highly abstract, self-referential skill — a question this project's prompt-ablation cells (9-20) are also probing from the learner side.

Agent Architecture

UniClawBench: A Universal Benchmark for Proactive Agents on Real-World Tasks WATCH

Zhekai Chen, Chengqi Duan, Kaiyue Sun, Bohao Li, Yuqing Wang, Manyuan Zhang, Xihui Liu · arXiv 2607.08768 · July 2026 · https://arxiv.org/abs/2607.08768

Argues existing agent benchmarks evaluate proactive agents in sandboxed, single-turn settings with scenario-based task taxonomies that mix several distinct capabilities inside one task category — which makes it impossible to tell which specific capability failed when an agent underperforms. UniClawBench instead structures evaluation around a capability-driven taxonomy in dynamic, multi-turn, real-world-style settings.

Project relevance: Only a loose fit — UniClawBench targets tool-using personal-assistant agents, not tutoring dialogue — but the underlying methodological point (capability-driven decomposition over scenario-mixed taxonomies, to localize *which* capability failed) is exactly the diagnostic gap the A13 trap-scenario suite (cells 111-113) and the adaptive runner's counterfactual replay (cell 110, services/adaptiveTutor/) are built to close for tutoring specifically. Worth a skim as a check on whether the trap suite's binary strict_shift scoring is itself scenario-mixing capabilities that a more decomposed taxonomy would separate.
