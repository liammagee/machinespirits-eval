Prompt Engineering

From Prompt Engineering to Epistemic Prompting: Prompt Trajectories as AI-Mediated Problem Framing in Science Education UNBLOCK

Matteo Tuveri · arXiv 2607.11680 · July 2026 · https://arxiv.org/abs/2607.11680

Reframes prompting in STEM education as a continuing epistemic practice rather than a one-shot technical skill, introducing a multi-turn "Framing-Prompting Loop" and the notion of a prompt framing trajectory: the sequence of prompts, model responses, learner uptake, disciplinary checks, and reframing moves through which a knowledge task develops across turns.

Project relevance: A close conceptual match for cell 21 (dynamic prompt rewriting with the Writing Pad) and the prompt-erosion cells 48-49: both track how the tutor's prompt evolves turn-over-turn as it incorporates learner state, which is exactly the "framing trajectory" this paper formalizes for a human tutor/learner pair. Its trajectory schema (prompt → response → uptake → check → reframe) is a candidate structured unit for services/dialogueTraceAnalyzer.js's superego-feedback-incorporation analysis, and offers external, education-research-grounded vocabulary for what "prompt evolution" should be measured against rather than only the project's own text-proxy erosion metrics.

Prompt engineering using order-of-addition experiments: An application to generating two-level fractional factorial designs WATCH

Duoduo Danny Ying, Alan R. Vazquez, Hongquan Xu · arXiv 2607.05537 · July 6, 2026 · https://arxiv.org/abs/2607.05537

Addresses LLM order dependency — performance varying with the order phrases appear in a prompt — by borrowing order-of-addition experimental design from statistics to systematically quantify ordering effects and locate the best position for each prompt element, demonstrated on generating fractional factorial designs.

Project relevance: The eval repo already runs a large factorial itself (cells 1-125), so the methodology is a natural fit conceptually, but the application domain is unrelated to tutoring. Background relevance to any future ablation of component order within the ego/superego prompts (e.g. where recognition-theory language or hardwired-rules blocks sit relative to task instructions in cells 13-14) rather than an immediate unblock.

Agent Architecture

Evidence-Grounded Verified Agentic Reasoning: A Path Toward Eliminating LLM Hallucination in Empirical Inference via Tool-Attested Kernel Proofs UNBLOCK

arXiv 2607.12650 · July 14, 2026 · https://arxiv.org/abs/2607.12650

EG-VAR is a Lean 4-based tool-calling architecture where the proof kernel, not the LLM, is the sole minter of "verified" claims: every accepted output must structurally descend from an attested tool call plus a kernel-checked inference chain, or else the system must honestly Abstain with a replayable audit trail. On TableBench numerical reasoning it hits 120/120 versus a 95% same-tool baseline; under counterfactual stress tests it stays 100% source-faithful while the same-tool baseline drops to 80-90%.

Project relevance: Directly on-target for the A13 pre-registration's C4 validator condition (cells 111-113, services/adaptiveTutor/) — EG-VAR is essentially a formalized, kernel-proof version of "programmatic constraints on the tutor's claims," which is what the validator condition approximates with hand-written checks. Its Abstain-over-hallucinate discipline is a plausible upgrade path for policyActions.js: instead of scoring whether the tutor's claim about learner state was validated post hoc, force claims to descend from an attested trace entry or block the turn. Also relevant to reducing false-positive "strategy_shift_correctness" hits in the trap-scenario suite (cells 110-125) if shift claims could be kernel-checked against the persisted state rather than judge-inferred.

Agora: Enhancing LLM Agent Reasoning Via Auction-Based Task Allocation WATCH

arXiv 2607.09600 · July 2026 · https://arxiv.org/abs/2607.09600

Treats individual reasoning steps as tradeable items in an incentive-compatible auction: expert models/tools bid on sub-tasks based on rectified (de-overconfidenced) competence estimates, routing critical logic to the most capable solver rather than the most confident one. Improves over matched routing/cascade baselines across five benchmarks with a controllable cost-quality trade-off.

Project relevance: Loosely adjacent to the id-director architecture (cells 101-109, services/idDirectorEngine.js), which already constructs a per-turn persona envelope but does not currently arbitrate between competing mechanism variants by anything like a competence-weighted bid. Background interest for a hypothetical future "mechanism router" across the self-reflective/quantitative/erosion variants in cells 40-59, rather than an immediate fit — the current architecture uses fixed cell assignment, not dynamic allocation.

Dramaturgy / Role-Play

Narrative World Model: Narratology-Grounded Writer Memory for Long-Form Fiction UNBLOCK

Mohammad Saifullah, Thomas Kornmaier, Taaha Kazi, Vasu Sharma, Aditya Sanjiv Kanade, Aanand Kumar Yadav · arXiv 2607.05577 · July 6, 2026 · https://arxiv.org/abs/2607.05577

Argues that general-purpose retrieval/agent-memory represents entities and facts but not the narratological structure long-form fiction turns on, and introduces the Narrative World Model (NWM): a typed temporal-state graph paired with query-conditioned hybrid retrieval, built to answer exactly the multi-hop questions that drive plot — who knows a secret and when they learned it, whether an event preceded the narration that revealed it, whether a setup paid off, and how a relationship shifted.

Project relevance: Strikingly close to two pieces of this project's own machinery: the withheld-secret S plus ordered premise ledger that /ms-play-tutor hand-tracks for Oedipus-style guided-discovery scenarios, and the peripeteia/anagnorisis structure that config/evaluation-rubric-poetics.yaml scores at the whole-transcript level. NWM's typed temporal-state graph is essentially a formalized data structure for "who knows what, when" and "did the setup pay off" — the same two questions the poetics rubric and the trap-scenario suite (cells 110-125) are trying to answer by LLM judgment alone. Worth reading for the adaptive tutor's state schema (services/adaptiveTutor/stateSchema.js): a narratology-typed graph could replace or audit the current ad hoc state tracking for counterfactual replay.

Rhetoric

Epistemic Stance Flexibility Probing: Measuring Prompt-Conditioned Register Shift in Large Language Models WATCH

Binwen Liu, Yilin Ren · arXiv 2607.12739 · July 14, 2026 · https://arxiv.org/abs/2607.12739

Introduces ESFP, a 104-item behavioral benchmark testing whether a model correctly distinguishes "what do experts believe about X" (neutral attribution register) from "what do you believe about X" (stance-expression register) across six epistemic categories. Finds epistemic flexibility is largely orthogonal to general capability — a 27B open-weight model matches frontier proprietary systems — and that surface markers like "I think" can shift without any corresponding change in actual expressed stance.

Project relevance: Relevant background for the charisma cells (101-109, config/evaluation-rubric-charisma.yaml), where register variants explicitly manipulate how directively/attributively the tutor speaks, and for the divergent-superego cells (22-33), where the superego's critique register (suspicious/adversary/advocate) is supposed to differ systematically from the ego's. ESFP's finding that lexical stance markers can decouple from actual expressed stance is a caution for any future rubric dimension that scores register from surface phrasing alone rather than from the stance-content-density signal the paper finds is the stronger discriminator.
