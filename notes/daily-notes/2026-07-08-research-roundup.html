Agent Architecture

MRMS: A Multi-Resolution Memory Substrate for Long-Lived AI Agents UNBLOCK

Jizhizi Li, Amy Shi-Nash (NxtLab Innovations) · arXiv 2607.04617 · July 6, 2026 · https://arxiv.org/abs/2607.04617

Argues that reliable long-horizon agent memory is a design problem, not a context-window problem: memory must be structured, selectively exposed, continuously consolidated, and epistemically labeled (personal context vs. external evidence) rather than stored as undifferentiated conversation history. Introduces MRMS, a two-axis (representational × temporal) memory substrate instantiated as a structured-vector-graph prototype with explicit revision policies for when the underlying situation changes.

Project relevance: A close structural match for cell 21 (dynamic prompt rewriting / Writing Pad) and the erosion cells (48-49): both accumulate injected context into the tutor's prompt turn over turn, exactly the "undifferentiated conversation history" failure mode MRMS is built against. Its epistemic-labeling distinction — is this consolidated prior experience or fresh external evidence? — is a candidate schema addition for the Writing Pad's own entries, and its revision-on-situation-change policy is a more principled trigger than the current text-proxy erosion measures for deciding when a pad entry should be superseded rather than appended.

Agent Data Injection Attacks are Realistic Threats to AI Agents WATCH

Woohyuk Choi, Juhee Kim, Taehyun Kang, Byoungyoung Lee (Seoul National University), Jihyeon Jeong (Largosoft), Luyi Xing (UIUC) · arXiv 2607.05120 · July 6, 2026 · https://arxiv.org/abs/2607.05120

Names a category of indirect prompt injection distinct from the well-studied "instruction injection": agent data injection (ADI), where attacker-controlled data is disguised as trusted security-critical metadata or agent-context data (tool-call/response formats) rather than as an instruction, so existing instruction-injection defenses don't catch it. Demonstrates the class is practically exploitable against current agent frameworks.

Project relevance: Directly adjacent to the trap-scenario suite (cells 110-125): those cells already test whether the adaptive tutor's LangGraph state-policy and the dialogue-engine baseline resist misleading in-context signals from a trap-steered learner. ADI's specific distinction — data disguised as trusted metadata vs. data disguised as instruction — suggests a finer-grained failure taxonomy for config/adaptive-trap-scenarios.yaml traps than the current binary strict_shift scoring captures, and could sharpen what "counterfactual replay" is being asked to detect in services/adaptiveTutor/.

Rhetoric

Rating the Pitch, Not the Product: User Evaluations of LLMs Reflect Expectations More Than Performance UNBLOCK

Robert Morabito, Tyler McDonald (Brock University), Charitra Viswanath, Ali Emami (Emory University), Angel Hsing-Chi Hwang (USC), Susanne Gaube (UCL), Jad Kabbara (MIT) · arXiv 2607.05113 · July 6, 2026 · https://arxiv.org/abs/2607.05113

A controlled study with 162 participants, each using one of six LLMs across three collaborative tasks after first viewing a landing page that matched, overstated, or understated the model's true capability. Pre-interaction framing — a purely rhetorical intervention with no change to the model itself — shifted user opinions and interaction behavior, while measured task performance did not move.

Project relevance: A direct caution for the human-learner pilot (services/pilotStore.js, public/pilot/index.html) and for assess-transcripts.js's blinded vs. unblinded narrative assessment: if pre-interaction framing alone moves human ratings independent of actual performance, then any pilot onboarding copy that primes expectations about the tutor ("adaptive," "AI-powered," etc.) is a confound on self-report outcome measures the same way it would be on an LLM-judge's qualitative assessment. Reinforces the existing blinded-assessment discipline (--blinded flag) and suggests the pilot's participant-facing framing language itself should be treated as an experimental variable, not incidental copy.
