Rohit Mehra, Kapil Singi, Vikrant Kaulgud, Vibhu Saujanya Sharma, Swapnajeet Gon Choudhury, Swati Sharma, Adam P. Burden, Majd Sakr · arXiv 2607.06074 · July 2026 · accepted, ASE '26 New Ideas & Emerging Results track · https://arxiv.org/abs/2607.06074

An empirical evaluation of "Prompt Coach," an agentic tutor purpose-built to teach software developers prompt-engineering skill rather than to answer coding questions directly — the tutor role is instructional coaching on a meta-skill, not task completion.

Agent Architecture

LLM Agents for Deliberative Collaboration: A Study on Joint Decision Making Under Partial Observability UNBLOCK

Chenxu Wang, Yongkun Yang, Boyuan Du, Shiwei Lin, Huaping Liu · arXiv 2607.06157 · July 2026 · cross-listed cs.CL / cs.AI · https://arxiv.org/abs/2607.06157

Studies how LLM agents should deliberate and reach a joint decision when each agent only has partial observability of the shared state — the setting is explicitly framed as agents holding incomplete, asymmetric views of a situation and needing structured deliberation to converge on a decision.

Project relevance: Structurally close to two live architectures at once. First, cells 110/111-113/124's adaptive runner (services/adaptiveTutor/) externalises learner state and drives a LangGraph state-policy over it — exactly a partial-observability setup, since the tutor agent never has ground truth on the learner's actual understanding, only inferred state. Second, the bilateral ego-superego architecture is itself a two-agent deliberation reaching one joint output (the ego's revision) from partially-shared views (superego critiques privately, ego revises publicly). A joint-decision-under-partial-observability framing is a candidate lens for a future process-measure alongside the existing RevΔ/AdaptΔ traces in analyze-mechanism-traces.js.

Memory-Orchestrated Semantic System (MOSS): An Auditable Agentic Memory Architecture WATCH

Serge Lacasse, Jérémie Hatier, Alex Baker · arXiv 2607.04391 · July 5, 2026 · https://arxiv.org/abs/2607.04391

Proposes an agentic memory architecture where the agent itself drives retrieval over a structured relational database rather than an embedding-similarity index — retrieval execution is symbolic and reproducible (no LLM in the retrieval loop once a query is formed), and every step from indexing to answer formulation is logged and inspectable. Reports a roughly year-long production deployment over one scholar's working corpus: 44M tokens, 110K+ segments, 569 inductively-derived concepts, ~5M relations across eleven metadata graphs.

Project relevance: A pointed contrast case for cell 21's dynamic prompt rewriting / Writing Pad and cells 48-49's prompt erosion: MOSS's core claim is that embedding-based similarity retrieval is "opaque by construction," and it substitutes an auditable, symbolic retrieval path instead. The Writing Pad's accumulated context and prompt_content_hash drift tracking currently rely on text-proxy measures with no equivalent auditability guarantee — MOSS's logged-and-inspectable retrieval trail is a design reference if the Writing Pad's memory mechanism is ever re-architected for provenance rather than just measured after the fact.
