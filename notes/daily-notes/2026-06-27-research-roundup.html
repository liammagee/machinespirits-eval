Prompt Engineering

FAPO: Fully Autonomous Prompt Optimization of Multi-Step LLM Pipelines UNBLOCK

Paul Kassianik, Baturay Saglam, Huaibo Zhao, Blaine Nelson, Supriti Vijay, Aman Priyanshu, Amin Karbasi · Foundation AI–Cisco Systems / Yale University · arXiv 2606.19605 · June 19–20, 2026 · https://arxiv.org/abs/2606.19605

FAPO diagnoses multi-step pipeline failures at the step level — retrieval, reasoning, formatting — attributes the bottleneck, proposes a scoped prompt or chain-structure edit, and validates the variant against a score function, all without human intervention. Across six benchmarks and three task models it beats the GEPA baseline in 15 of 18 comparisons with a mean gain of +14.1 percentage points. The key insight is that blind full-pipeline prompt tuning misfires when failure is structural: FAPO upgrades to a structural fix only when prompt edits are provably insufficient.

Project relevance: The npm run prompt-lab -- autotune workflow currently relies on human-directed iteration over prompts/tutor-ego*.md. FAPO's step-level attribution + autonomous loop is a drop-in pattern for this: treat each turn of the tutor pipeline (ego generate → superego critique → ego revise) as a named stage, attribute low tutor_first_turn_score returns to the correct stage, then mutate only that stage's prompt. Especially valuable for cells 93–100 (superego variant ablations) where the failing stage is already known but the correct prompt edit is not.

Agent Architecture

A Technical Taxonomy of LLM Agent Communication Protocols WATCH

Linus Sander, Habtom Kahsay Gidey, Alexander Lenz, Alois Knoll · Technische Universität München · arXiv 2606.19135 · June 2026 · https://arxiv.org/abs/2606.19135

The paper constructs a five-dimension taxonomy — counterparty type, payload format, interaction-state persistence, discovery mechanism, and schema flexibility — by running five grounded-theory iterations over nine actively maintained open-source agent-to-agent protocols. The dominant pattern across all sampled protocols is hybrid payload + session-state persistence; pure-text or fully stateless protocols are vanishingly rare in production systems.

Project relevance: The tutor-core ego/superego exchange and the learner deliberation cycle are both multi-step agent communication patterns. Mapping them onto this taxonomy reveals the current architecture is hybrid-payload + session-state (the dialogue log) — consistent with production norms — but the discovery mechanism is hardwired (single-host, no registry). The taxonomy is useful scaffolding if the project ever externalises the adaptive runner's state-policy into a separable microservice; the interoperability gaps the paper identifies would surface immediately.

Sakana Fugu Technical Report: A Trained Orchestrator for Multi-Agent LLM Systems WATCH

Fugu Team, Sakana AI · arXiv 2606.21228 · submitted June 19, released June 22, 2026 · https://arxiv.org/abs/2606.21228

Fugu is a language model trained to orchestrate a pool of other LLMs (Claude Opus 4.8, GPT-5.5, Gemini 3.1 Pro, including recursive self-calls) by dynamically devising agentic scaffolds — selecting which model to call, with what context, verified by what critic — for each sub-task. The orchestrator is itself a trained model, not a hardwired policy: it achieves state-of-the-art results on SWE-Bench Pro, GPQA-Diamond, and eight additional benchmarks, beating every member of its pool individually.

Project relevance: The LangGraph adaptive runner (cells 110–113) already implements a learned state-policy that selects among pedagogical moves — a structural analogue of Fugu's model-selection loop. Fugu extends this idea to cross-model orchestration: the ego/superego split could be implemented as a Fugu-style dispatch (ego = Haiku for speed, superego = Opus for depth) with the orchestrator trained on rubric returns rather than benchmark accuracy. The paper's "Learning to Orchestrate Agents" lineage (ICLR 2026) is worth reading alongside the adaptive runner design in services/adaptiveTutor/.

Ontology / Knowledge Representation

Specific Domain Ontology Construction Using Large Language Models WATCH

arXiv 2606.20691 · June 2026 · https://arxiv.org/abs/2606.20691

The paper uses LLMs (GPT-3.5, GPT-4) in the role of domain experts to iteratively build conceptual hierarchies for a target concept, producing 20 ontologies for the Brazilian maritime territory (the "Blue Amazon"). Human expert evaluation finds the outputs overall coherent but not satisfactory without refinement — LLMs correctly populate mid-level classes but struggle with precise boundary conditions and rare sub-concepts. The key contribution is the prompt architecture for role-assignment and iterative hierarchy refinement.

Project relevance: The poetics ontology (config/evaluation-rubric-poetics.yaml) is currently a flat YAML with six Aristotle-derived dimensions. The κ ≈ 0.04 critic-divergence failure suggests critics lack shared ontological grounding. This paper's LLM-as-domain-expert approach could bootstrap a richer hierarchy for peripeteia / anagnorisis / hamartia — treating each concept as the root of a subgraph rather than a flat rubric dimension. Combine with the KG-grounding pattern from 2606.11560 (prior note) for a constrained-scoring step that replaces free-text rubric descriptions.

Dramaturgy / Narrative AI

AI Fiction in the Wild: Roleplay, Fanfiction, and Infinite Story Demanders WATCH

Neel Gupta, Maria Antoniak, Melanie Walsh · arXiv 2606.22748 · June 22–23, 2026 · https://arxiv.org/abs/2606.22748

Drawing on 500,000+ anonymised ChatGPT conversations, the study finds that more than one third involve fiction generation — original stories, roleplay, fanfiction, or erotica. Usage is heavily concentrated among power users ("infinite story demanders") who repeatedly request and revise similar narratives across extended sessions. Stylistically, AI-generated fiction gravitates toward generic forms, repetition, immediacy, and niche combinatorics, diverging substantially from human literary norms.

Project relevance: The drama machine and poetics pipeline generate tutoring dramas, but they operate without knowing what makes people return to LLM fiction or what satisfies them. This paper provides the empirical baseline: users prefer immediacy and genre familiarity over literary quality. The "infinite story demander" profile maps onto learners who replay the same tutoring scenario in slightly different framings — a realistic usage mode the drama generator should support explicitly. Also relevant to the §6.3 suggestion-scenario design: the "niche combinatorics" finding suggests learners probe edge cases spontaneously, not just canonical examples.
