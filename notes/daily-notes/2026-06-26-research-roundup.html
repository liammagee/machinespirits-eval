Agent Architecture & Memory

Multi-Agent Transactive Memory WATCH

To Eun Kim, Xuhong He, Dishank Jain, Ambuj Agrawal, Negar Arabzadeh, Fernando Diaz (Carnegie Mellon University; UC Berkeley) · arXiv 2606.19911 · submitted 18 Jun 2026 (announced 19 Jun) · https://arxiv.org/abs/2606.19911

MATM extends retrieval-augmented generation to the population level: producer agents contribute their full interaction trajectories to a shared repository; consumer agents query it at inference time to import procedural knowledge without re-experiencing the task themselves. Trajectories — currently discarded after single use or siloed per agent — become a durable, queryable commons. Evaluated across heterogeneous agent types on diverse task suites from CMU and Berkeley.

Project relevance: The tutor-ego/learner-ego pair already accumulates dialogue logs in exports/ and EVAL_LOGS_DIR; those logs are effectively discarded after scoring. MATM points toward a retrieval layer where a new tutoring run could query successful prior dramas as exemplars — a concrete path for implementing the "other-ego profiling" mechanism of cells 54–59 at population scale. Also directly relevant to the Drama Machine's multi-agent ensemble, where narrator, tutor, learner, and judge agents could share trajectory history.

ESAA-Conversational: Event-Sourced Memory for Continuity and Handoff Across Heterogeneous LLM Agents WATCH

Elzo Brito dos Santos Filho et al. · arXiv 2606.23752 · submitted 22 Jun 2026 (announced 23 Jun) · https://arxiv.org/abs/2606.23752

An event-sourced architecture that treats the visible conversation as a local append-only event store (activity.jsonl). Hooks and watchers normalize each agent's turns into structured activity events; deterministic read-model projections then derive handoff.md, state.md, decisions.md, and tasks.json views — enabling reliable state transfer when a different agent (or model) takes over mid-session, regardless of provider or tool differences.

Project relevance: The bilateral ego-superego pipeline writes dialogue logs turn-by-turn but has no formal handoff contract for when a run resumes, a judge replaces a generator, or the desktop app re-opens a session. ESAA's append-only log + derived views is a natural fit for services/adaptiveTutor/persistence.js and the poetics ingest pipeline, where transcript state needs to survive provider switches (Sonnet → Opus → GPT judge) without losing which turns were already scored.

Prompt Engineering & Pedagogical AI

Learning to Prompt: Improving Student Engagement with Adaptive LLM-based High-School Tutoring UNBLOCK

Po-Chin Chang, Nicholas Hogan, Aske Plaat, Michiel T. van der Meer · arXiv 2606.20138 · submitted 18 Jun 2026 (announced 19 Jun) · https://arxiv.org/abs/2606.20138

Extracts 14 pedagogical features (tutor scaffolding level, student understanding markers, question type, etc.) from raw multi-turn transcripts and trains a prompt-routing model in simulation that switches between an "analytical" and a "scaffolding" prompt strategy at each turn. Deployed in an A/B test with 656 real high-school conversations; shows measurable sim-to-real transfer — the routing model generalises from synthetic learners to actual students with improved engagement metrics.

Project relevance: This is a direct implementation of the adaptive strategy-switching that cells 110–113 target at the architecture level. The 14-feature transcript extractor is a candidate drop-in for the analyze-within-test-change.js pipeline: instead of computing a single tutor_development_score, extract per-turn pedagogical features and let the routing model score strategy coherence across the drama arc. The sim-to-real methodology also validates the project's approach of training on synthetic (LLM-learner) transcripts before human-learner validation.

Ontology / Knowledge Representation

Toten: Knowledge-Based Ontological Tokenization of Physical Quantities and Technical Notation WATCH

Researchers on ontological NLP (Brazilian Portuguese corpus) · arXiv 2606.19626 · submitted 18 Jun 2026 (announced 19 Jun) · https://arxiv.org/abs/2606.19626

Replaces statistical byte-pair encoding with a declarative classifier grounded in the Ontology of Engineering Entities (OEE). Physical quantities, measurement units, numbers, and symbolic notation are fragmented according to formal ontological categories rather than lexical frequency — producing tokens that carry semantic type information as a first-class property. Applied to Brazilian Portuguese scientific text but designed as a language-agnostic framework.

Project relevance: The poetics ontology (peripeteia, anagnorisis, hamartia, etc.) is currently an informal YAML schema and a free-text rubric. Toten's approach — replacing statistical segmentation with ontology-grounded type categories — is a concrete design pattern for promoting config/evaluation-rubric-poetics.yaml to a structured type system: each dramatic event in a transcript could be tagged with an OEE-style ontological category that the critic then uses as a scoring anchor, directly addressing the κ ≈ 0.04 inter-rater failure from underconstrained free-text rubric application.
