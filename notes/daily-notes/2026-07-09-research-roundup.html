Prompt Engineering

MRMS: A Multi-Resolution Memory Substrate for Long-Lived AI Agents UNBLOCK

Jizhizi Li, Amy Shi-Nash (NxtLab Innovations) · arXiv 2607.04617 · July 6, 2026 · https://arxiv.org/abs/2607.04617

Argues that continuity for a long-lived agent cannot come from simply extending the prompt window: an agent must preserve useful prior experience, retrieve it selectively, distinguish personal context from external evidence, and revise memory when the underlying situation changes. Central claim — reliable personalization is a memory-design problem, not a retrieval problem — is operationalized as MRMS, a prototype combining structured records, vector retrieval, temporal policies, and graph-based revision.

Project relevance: A ready-made design vocabulary for cell 21 (dynamic prompt rewriting with the Writing Pad) and cells 48-49 (prompt erosion): both mechanisms accumulate injected context into the tutor's prompt turn over turn, exactly the "undifferentiated conversation history" MRMS argues against. MRMS's four properties — structured, selectively exposed, continuously consolidated, epistemically labeled — give the Writing Pad a principled target state, and its temporal-policy/graph-revision split is a plausible complement to the text-proxy erosion measures already used to score cells 48-49: instead of only measuring how much drift has occurred, it suggests measuring whether the Writing Pad is consolidating (MRMS-like) or merely concatenating (erosion-prone).

Agent Architecture

LLM-as-a-Verifier: A General-Purpose Verification Framework UNBLOCK

Jacky Kwok, Shulu Li, Pranav Atreya, Yuejiang Liu, Yixing Jiang, Chelsea Finn, Marco Pavone, Ion Stoica, Azalia Mirhoseini · arXiv 2607.05391 · July 6, 2026 · https://arxiv.org/abs/2607.05391

Identifies verification — determining whether a candidate solution is correct — as a distinct scaling axis alongside pre-training, post-training, and test-time compute. Unlike standard LM-judge prompting, which produces a single discrete score, LLM-as-a-Verifier computes the expectation over the distribution of scoring-token logits to produce a continuous score, which the authors show scales along three dimensions: score granularity, repeated evaluation, and criteria decomposition. State-of-the-art results on Terminal-Bench V2, SWE-Bench Verified, RoboRewardBench, and MedAgentBench.

Project relevance: Directly bears on the v2.2 rubric judge pipeline (scripts/eval-cli.js evaluate and the dimension-scoring services): today's judge calls elicit a single discrete score per dimension per rubric version, the exact "standard LM judge" pattern the paper contrasts itself against. Logit-expectation scoring is a candidate drop-in for reducing judge-side score quantization noise — relevant to analyze-judge-reliability.js's inter-judge correlation work, since a continuous score would give correlation analyses more signal than the current discrete 1-N buckets, and the "criteria decomposition" axis maps naturally onto the 8-dimension v2.2 rubric's per-dimension scoring.

Agent Data Injection Attacks are Realistic Threats to AI Agents WATCH

Woohyuk Choi, Juhee Kim, Taehyun Kang (Seoul National University), Jihyeon Jeong (Largosoft), Luyi Xing (University of Illinois Urbana-Champaign), Byoungyoung Lee (Seoul National University) · arXiv 2607.05120 · July 6, 2026 · https://arxiv.org/abs/2607.05120

Introduces agent data injection (ADI), a category of indirect prompt injection distinct from instruction injection: rather than smuggling in new instructions, ADI disguises malicious data as trusted metadata the agent already treats as authoritative — resource identifiers, data origins, or tool call/response formats — causing the agent to take unintended actions on attacker-controlled data it never questions as data.

Project relevance: A relevant threat model for the adaptive runner's structured state (services/adaptiveTutor/ stateSchema.js, policyActions.js) and the id-director's persona envelope (id_construction_trace): both architectures feed structured, agent-trusted context (learner state, id-persona JSON) into downstream policy/generation steps much like the "agent context data" ADI targets. Not an immediate action item — the project's agents don't currently ingest untrusted external data — but worth keeping on the radar if the adaptive trap-scenario suite (cells 110-113, 124) is ever extended to ingest externally-sourced scenario content.
