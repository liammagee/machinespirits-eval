Prompt Engineering

FAPO: Fully Autonomous Prompt Optimization of Multi-Step LLM Pipelines UNBLOCK

Paul Kassianik, Baturay Saglam, Huaibo Zhao, Blaine Nelson, Supriti Vijay, Aman Priyanshu, Amin Karbasi · Cisco Foundation AI · arXiv 2606.19605 · June 17–18, 2026 · https://arxiv.org/abs/2606.19605

FAPO is an autonomous agent that iteratively evaluates a multi-step LLM pipeline, inspects intermediate outputs, diagnoses failures, proposes scoped prompt or structural revisions, then validates each variant against a score function — no human input required. Across six benchmarks and three task models, FAPO outperforms the previous best automated optimiser (GEPA) in 15 of 18 comparisons with a mean gain of +14.1 percentage points. It closes the loop on pipeline-level prompt engineering entirely.

Project relevance: The npm run prompt-lab -- autotune workflow is the project's analogue. FAPO's intermediate-output inspection step is exactly the gap the current autotune lacks: it knows the final tutor_first_turn_score but not which module (ego generation vs. superego critique vs. ego revision) caused the shortfall. FAPO's module-attributing architecture could be ported to decompose the ego → superego → revision chain and score each stage independently — directly unblocking the prompt-lab pipeline for cells 93–100 (superego variant ablations).

Agent Architecture

Multi-Agent Transactive Memory UNBLOCK

To Eun Kim, Xuhong He, Dishank Jain, Ambuj Agrawal, Negar Arabzadeh, Fernando Diaz · Carnegie Mellon University & UC Berkeley · arXiv 2606.19911 · June 18, 2026 · https://arxiv.org/abs/2606.19911

MATM (Multi-Agent Transactive Memory) borrows the cognitive-science concept of transactive memory — groups distribute knowledge by delegating ownership and maintaining shared retrieval pointers — and applies it to heterogeneous LLM agent populations. Producer agents contribute solved task trajectories to a shared repository; consumer agents retrieve and reuse them rather than rediscovering solutions. The key argument: agent-generated procedural knowledge is currently discarded after single use, forcing repeated rediscovery across agent instantiations.

Project relevance: The ego-superego deliberation cycle already produces rich artifact traces (scores_with_reasoning, tutor_deliberation_*, learner_deliberation_*) stored in the DB but never reused at inference. MATM suggests a retrieval layer: past superego critiques that matched a scenario can seed the current superego's prior, making revision more principled. This also directly addresses Finding 11 / D3 (insight-action gap) — agents that extract insight but don't act on it might do so if prior successful revisions were retrievable.

Ontology / Knowledge Representation

Per-Entity Bias Mapping for AI Visibility: Why Brand Mentions Require Entity-Specific Calibration WATCH

Zoltan Varga (independent) · arXiv 2606.21595 · June 19, 2026 · https://arxiv.org/abs/2606.21595

Introduces PEBM (Per-Entity Bias Mapping), a ten-dimension audit framework for how AI answer systems represent entities. Three structural failure modes are identified: invisibility (weak knowledge-graph presence), the Brand Hallucination Paradox (familiarity breeds confident errors for popular entities), and a geographic coverage gap in KG entity linking for Central-Eastern European entities. The core argument is that aggregate mention-rate metrics mask systematic entity-specific failure modes that only per-entity calibration can surface.

Project relevance: The poetics ontology (peripeteia / anagnorisis / hamartia / etc.) is currently a flat YAML schema. PEBM's per-entity calibration idea translates here: different dramatic concepts have different coverage in training corpora and therefore different confidence calibration under the critic LLM. The "Brand Hallucination Paradox" is directly observable in poetics scoring — anagnorisis may be confidently over-applied precisely because it is a prominent concept the model has seen many times, while rarer concepts like hamartia are under-detected. A per-concept calibration audit of the poetics critic would be a concrete near-term fix for κ divergence.

Dramaturgy / Role-play

DramaDirector: Geometry-Guided Short Drama Generation WATCH

Hengji Zhou, Sijie Liu, Jianrun Chen, Xingchen Zou, Liqiang Nie et al. · arXiv 2606.24107 · June 23, 2026 · https://arxiv.org/abs/2606.24107

DramaDirector addresses plot-to-video short-drama generation by decoupling each shot into static visual conditions and dynamic narrative conditions. A planner is trained with schema-constrained SFT and GRPO under a learned text-visual alignment reward; real-shot depth-pose references guide video synthesis. LLMs act as narrative planners within a cinematographic geometry framework — the narrative structure must translate to concrete enacted scenes with geometric consistency.

Project relevance: The scripts/drama-generator.js and poetics pipeline face a structurally analogous problem: converting an abstract dramatic plan (peripeteia arc, anagnorisis moment) into a concrete enacted transcript. DramaDirector's schema-constrained planning approach — where an LLM planner must satisfy a structural schema before generating — is a direct design pattern for adding arc-consistency constraints to the drama generator. The GRPO reward training is also relevant to the adaptation-loop: if the loop currently relies on free-form critic feedback, a geometry-analogous reward signal (e.g., "does the generated drama actually contain a recognition moment?") could make training more targeted.

Rhetoric / Argumentation

Age of LLM: A Strategic Benchmark for Reasoning, Diplomacy and Reliability under Fog of War WATCH

Arnaud Ricci (independent, Switzerland) · arXiv 2606.24391 · June 23, 2026 · https://arxiv.org/abs/2606.24391

Introduces a turn-based 1v1 game benchmark on a 13×7 grid with three deliberate stressors: fog of war (incomplete information), full diplomacy (natural-language ceasefires, ultimatums, resource-hiding), and strict JSON schema compliance. 15 LLMs were tested across 54 matches and 5,258 actions. The diplomacy dimension operationalises rhetorical and argumentative behaviour in adversarial dialogue with a private engine and random seeds to prevent data contamination.

Project relevance: The tutoring dialogue shares structural properties with adversarial diplomacy: both involve incomplete information (the tutor doesn't know the full extent of the learner's misconception) and rhetorical pressure (the learner resists revision). The diplomacy dimension's natural-language ultimatums are especially close to the adaptive runner's trap scenarios (cells 110–113), where the tutor must recognise and respond to a strategic misrepresentation. The benchmark's strict schema compliance requirement is also a clean proxy for the passes_required / passes_forbidden validation columns already in the DB.
