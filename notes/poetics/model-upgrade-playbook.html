How to upgrade weaker models into tutors

Evidence-bounded advice for spending prompts, critics, guards, and stronger models only where they buy real tutoring reliability.

The current answer is not that a cheap model becomes a frontier tutor by adding mechanisms. The stronger, safer claim is narrower: a weaker or cheaper model can often be moved up one useful tier if the tutor's stance is calibrated, its brittle decisions are moved into a critic or a proof guard, and its prompts are fitted to the model rather than copied from a stronger one. The mechanisms substitute for raw capability in specific channels: orientation, error correction, release timing, and role governance. They do not substitute for all capability everywhere.

00·

How to read this

This is a practical recipe, not a deployment guarantee.

Every recommendation below is tagged by evidential weight. The strongest claims inherit from docs/research/paper-full-2.0.md; the cost-aware routing claims inherit from a single local engineering diagnostic; the proof-DAG claims inherit from formal derivation worlds rather than LLM-judge tutoring scores.

!

Bound the claim before using the playbook important

Say: mechanisms can reduce dependence on raw model capability in bounded tutoring channels. Do not say: a cheap model becomes equivalent to an expensive model. The evidence supports approximation only relative to unassisted baselines or to stronger-model baselines with fewer mechanisms. It does not show that cheap+mechanisms matches strong+mechanisms.

Strongest evidence

  • Intersubjective-pedagogy calibration lifts tutor output across three generation models and three judges.
  • A critic helps when the base model still leaves corrigible failures.
  • Prompt complexity must be model-stratified.

Bounded evidence

  • Proof-DAG guards improve derivation conduct where task geometry matches the guard channel.
  • Mixed model routing can reduce metered API use in one diagnostic run.
  • Selector rules are low-regret, not solved adaptivity.

Not shown

  • No human-learner outcome validation yet.
  • No broad cost-per-learning curve.
  • No proof that very weak models recover from recognition alone.

discipline The page is advice from evidence, not new evidence. Re-check the source runs before turning any line into a paper claim.

01·

Upgrade ladder

Spend mechanisms in this order.

1. Calibrate the stance. Use an intersubjective / constructivist tutor prompt first. This is the cheapest and most replicated lift.
2. Add a critic only if failures remain. For weaker models, an ego → superego → ego-revision loop can be worth the token tax. For stronger calibrated models, it is often redundant.
3. Fit prompt weight to model capability. Weak models may need explicit diagnostic rules and examples. Strong models need light permission, not a cage of instructions.
4. Use proof guards for formal derivation tasks. Move release timing and proof debt into a runtime guard when correctness depends on an authored reasoning path.
5. Route roles by risk. Cheap models can carry fluent ego roles; spend stronger models or non-metered routes on director, critic, and guard-sensitive choices.
6. Measure before promoting. Use the same scenarios, judges, rubrics, and cost accounting. A nice transcript is not evidence of interchangeability.

rule Spend the cheapest mechanism that targets the failure channel you can actually observe.

03·

First mechanism: calibration

Upgrade the model's stance before its architecture.

Start with a system prompt that treats the learner as an active thinking subject: notice what is valid in the learner's attempt, preserve productive difficulty, ask diagnostic questions, and repair rupture before proceeding. The active ingredient is the intersubjective-pedagogy orientation; matched constructivist prompts reproduce the effect, while behaviorist specificity backfires.

1

Use calibration as the default cheap-model upgrade supported

The recognition / constructivist orientation effect is the most replicated finding: directionally unanimous across the three generation models and three judges, with pooled effect around d ≈ 1.63. It lifts floors, especially on weak baseline behavior. For deployment, this means the first dollar should go into a well-authored tutor prompt, not a second agent.

Implementation test A calibrated weak-model tutor should stop lecturing by default. It should ask content-bearing questions, use the learner's exact contribution, and make correction feel like a continuation of the learner's thought rather than a reset.

paper §6.1, §6.4, §7.9; active ingredient is orientation family, not Hegelian keywords by themselves.

04·

Second mechanism: critic

Use a critic as a safety net, not as decoration.

The critic pattern is ego draft → superego review → ego revision, with the ego retaining final authorship. It is worth paying for when it catches failures the base model cannot self-correct. It is wasteful when the calibrated model already avoids those failures.

2

Keep the critic for weak calibrated models with residual failures model-dependent

Under base prompts, the critic adds large gains; under recognition prompts, its residual value collapses on stronger models and persists on Gemini Flash. In the messages-mode comparison, recognition + critic takes Gemini Flash to 69.6, while recognition single is 53.5. For DeepSeek and Haiku, recognition + critic is flat or lower than recognition single in the same aggregate. Use the critic where residual failures remain, not because a two-agent diagram looks safer.

$

Budget implication operational

A 10-turn multi-agent dialogue is expensive because every public turn hides multiple draft, critique, and revision calls. If recognition single gets the weak model above the quality floor, stop there. If it still produces lecture dumps, premature closure, or affect-blind correction, pay for the critic on those turns or those domains.

paper §6.2 and §6.4: error correction is real, but partly substituted by calibration.

05·

Model fit

Weak models need scaffolds; strong models need permission.

The prompt-lab evidence is small-N but practically sharp. A weak local Qwen model improved when the prompt added explicit diagnostic probes, anti-patterns, examples, and superego intervention rules. Haiku degraded under similarly heavy treatment and improved only after a light, permission-oriented edit.

3

Maintain separate prompt tiers by capability bounded but actionable

On the targeted prompt-optimization scenario, Qwen 3.5 9B moved from 35.0 target-dimension score to 75.0 with heavy scaffolding. Haiku started at 90.0, was harmed by heavy rule additions (73-85), and reached 95.0 only with a light prompt. Cross-transfer was destructive: Qwen's heavy prompt on Haiku scored 62.2, below Haiku's own baseline.

Weak / cheap tier

  • Explicit diagnostic probes.
  • Concrete anti-patterns.
  • Worked examples.
  • Superego checks tied to known failure modes.

Stronger tier

  • Short identity and stance.
  • Light tracking heuristic.
  • Permission to vary strategy.
  • No bulky transferred rulebook.

paper §7.8.1-§7.8.4; the result is model-specific and single-scenario, but the transfer failure is exactly the warning builders need.

06·

Formal tasks

For derivation tutoring, upgrade conduct with a proof-DAG guard.

If the target lesson has a checkable reasoning path, do not rely on a weak model's spontaneous pacing. Externalize the proof state, maintain proof debt, and let a runtime guard decide whether to withhold, repair, or advance. This upgrades the conduct channel: timing, premise release, and proof continuity.

4

Use hidden proof-state guards when the page is not enough geometry-bounded

In linear worlds, visible page-only pacing can match proof-state pacing. In forked / AND-join worlds, page-only pacing fails where hidden proof-state succeeds. In distractor worlds, the reverse can happen. The robust practical default is hidden proof debt for production derivation runs, with visible-only guards reserved for worlds where visible uptake faithfully projects the hidden task state.

paper §6.13.11-§6.13.15; hidden+proofDebt is the production derivation arm, overlays remain research instrumentation.

07·

Role routing

Put the cheap model where fluency is enough.

Role routing is the practical bridge from "mechanisms help" to "spend less." The current diagnostic split put GLM 5.2 in tutor and learner ego roles while Codex carried director and superego roles. It reduced metered API use compared with all-GLM and did not collapse one scored transcript, but it was slower than Codex-only and remains n=1.

5

Route by responsibility diagnostic

Use cheaper models for high-volume ego drafting when the prompt is slim enough and failures are catchable. Use stronger or non-metered routes for director, critic, and proof-state decisions. In the AF1 diagnostic, mixed GLM ego + Codex governance used 26 API calls plus 14 Codex calls and cost $0.389746 in metered API use, versus all-GLM 40 API calls and $0.598107. Both Codex-only and mixed-role outputs received the same overall quality score in that one transcript. Treat this as a routing smoke, not a result.

Practical next test Repeat the three-way comparison after prompt slimming: Codex-only, all-GLM, and GLM ego + Codex governance. Hold lesson, seed, scorer, max turns, and prompt mode fixed; record retries and tail latency.

08·

What not to build

Do not mistake more machinery for an upgrade.

×

Skip bidirectional learner-profiling prostheses not supported

The cognitive-prosthesis hypothesis predicted that explicit learner profiling would compensate for weak ego reasoning. The A3 threshold report does not support that: five of six models showed significant harm, one showed null effect, and none showed significant benefit. Prompt-level scaffolding can help weak models; superego-routed bidirectional profiling is a different mechanism and currently a bad default.

×

Do not transfer the weak-model prompt upward harmful transfer

A heavy prompt that functions as a prosthesis for a weak model can become a straitjacket for a stronger one. Keep the model tier attached to the prompt artifact. If the routing layer swaps the model, it should swap the prompt tier too.

failure class The common error is to add state, memory, and profiling when the actual failure is poor orientation or missing proof-control.

09·

Promotion gate

A weak-model tutor is promoted only when the evidence moves with it.

Gate 1

Same-row quality

Compare the candidate to the stronger baseline on the same scenarios, same scorer panel, and same rubric version.

Gate 2

Failure channel audit

Check lecture dumps, premature closure, affect-blind correction, hallucinated derivation steps, and prompt leaks. Do not accept mean-score parity with worse tails.

Gate 3

Cost and latency accounting

Count all hidden calls: ego, critic, retries, judge, and guard overhead. A cheaper model plus many hidden repair calls can lose the cost advantage.

Gate 4

Scope statement

Name what the upgrade proves: tutor-output quality, derivation conduct, role-routing feasibility, or learner outcome. Only the first two have meaningful support here.

The target is not "cheap model equals expensive model." The target is: every expensive capability is either removed, routed, guarded, or paid for deliberately.practical reading of Paper 2.0 and the derivation/routing notes
